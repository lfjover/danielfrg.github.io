Machine Learning

Time to see if boosting and bagging are that good as they promise to be. We create a new machine learning instance and set the dataset and we tell them to sample half for training and half for testing.

In [4]:
ml = copper.MachineLearning()
ml.dataset = ds
ml.sample(0.5)

Create a new Decision Tree, add it to the models to compare and fit the models

In [5]:
from sklearn import tree
tree_clf = tree.DecisionTreeClassifier(max_depth=10)
ml.add_model(tree_clf, 'Decision Tree')
ml.fit()

Since I start coding this library I wanted to maintain all compatibility with pandas and scikit-learn and its respective API. For that reason is possible to just add already fitted classifiers to the ML class and then just compare them with the utilities; this is useful in the case of using bootstraping.

In the next lines I create 5 different Decision Tree classifiers using 5 different samples (using Bootstrap), fit each classifier and add them to the ML class.

In [6]:
from sklearn import cross_validation
bs = cross_validation.Bootstrap(len(ds.inputs.values), n_iter=5)
i = 0
for train_index, test_index in bs:
    X_train = ds.inputs.values[test_index]
    y_train = ds.target.values[test_index]
    clf = tree.DecisionTreeClassifier(max_depth=10)
    clf.fit(X_train, y_train)
    ml.add_model(clf, "DT" + str(i + 1))
    i += 1

Bagging

Since scikit-learn does not have a implementation of bagging. I made a really simple (and un-efficient) implementation of bagging by using the mode to predict the class and using the mean to predict the probabilities. Finally create some methods to make that possible and easy on the ML class.

But first I want to remove the first Decision Tree model, because it sucks.

Conclusion

Bootstraping and bagging are very good. Just a simple implementation gave much better results, also I only use 5 DT on the bag probably is a good idea to increase that number.

I did not select any parameter for the Decision Trees, only the max-depth=10, playing with more models and more parameters I am sure bagging is going to be even better. To solve this problem next I want to take a look at Grid Search and see how it can help me to improve this results.
